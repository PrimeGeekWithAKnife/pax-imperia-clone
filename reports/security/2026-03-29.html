A comprehensive security audit was performed across all three packages (server, client, shared). The server package (packages/server/src/) was the primary focus as it handles external input. No critical vulnerabilities such as eval(), SQL injection, command injection, or XSS via innerHTML were found in the TypeScript source. The main findings were: missing security headers, insufficient input validation on Socket.io event handlers, no WebSocket rate limiting, prototype pollution risk in the game:action payload, and hardcoded credentials in CLAUDE.md. All high-severity code issues have been fixed. Three npm dependency vulnerabilities were auto-fixed; five moderate dev-only vulnerabilities remain (esbuild/vite chain requiring a breaking major version upgrade).

Fixes Applied

1. Security Headers via @fastify/helmet

Vulnerability: Missing Content-Security-Policy, X-Content-Type-Options, X-Frame-Options, and other standard security headers.

Change: Installed @fastify/helmet and registered it with strict CSP directives (default-src ‘self’, no frames, WebSocket connections allowed, inline styles allowed for Phaser).

Files: packages/server/package.json, packages/server/src/main.ts

2. Socket.io Per-Socket Rate Limiting

Vulnerability: No rate limiting on WebSocket events allowed unlimited event flooding (DoS vector).

Change: Implemented a lightweight per-socket rate limiter using WeakMap with 60 events per 10-second sliding window. All Socket.io event handlers are wrapped via withRateLimit(). Throttled clients receive a structured error callback.

Files: packages/server/src/network/socketManager.ts

3. Prototype Pollution Protection for game:action

Vulnerability: The game:action handler accepted arbitrary nested objects that could contain __proto__, constructor, or prototype keys, enabling prototype pollution.

Change: Added sanitiseObject() function that recursively strips dangerous keys from arbitrary objects, with a maximum depth of 10 to prevent stack exhaustion.

Files: packages/server/src/network/sanitise.ts, packages/server/src/network/socketManager.ts

4. Species ID Validation

Vulnerability: The lobby:species handler passed speciesId directly without any type checking or length validation.

Change: Added sanitiseSpeciesId() with type checking, control character stripping, and 60-character length limit.

Files: packages/server/src/network/sanitise.ts, packages/server/src/network/socketManager.ts

5. Galaxy Config Enum Validation

Vulnerability: galaxyConfig.size and galaxyConfig.shape were not validated against their allowed enum values, allowing arbitrary strings to propagate through the system.

Change: Added sanitiseGalaxyConfig() that validates size against {small, medium, large, huge} and shape against {spiral, elliptical, irregular, ring}, falling back to defaults for invalid values.

Files: packages/server/src/network/sanitise.ts, packages/server/src/network/socketManager.ts

6. Action Type Validation

Vulnerability: The actionType field in game:action was not validated for type or length.

Change: Added type checking and 100-character length limit for actionType strings.

Files: packages/server/src/network/socketManager.ts

7. Dependency Vulnerability Fixes

Vulnerability: npm audit identified 8 vulnerabilities (1 high, 7 moderate).

Change: Ran npm audit fix which resolved 3 vulnerabilities (fastify, picomatch, brace-expansion). The remaining 5 moderate vulnerabilities are dev-only (esbuild/vite/vitest) and require a breaking vite@8.x upgrade.
