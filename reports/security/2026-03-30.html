Secure Patterns Observed

  • Input sanitisation: All player-supplied strings sanitised via sanitise.ts (control char stripping, length truncation)
  • Prototype pollution guard: sanitiseObject() strips __proto__, constructor, prototype keys recursively
  • Path traversal protection: Save file names use path.basename() + startsWith() validation
  • Rate limiting: Global HTTP rate limit (100/min/IP) and per-socket WebSocket rate limit (60/10s)
  • Security headers: Helmet configured with CSP, X-Frame-Options, X-Content-Type-Options
  • No eval/Function: Zero instances of eval() or new Function() across the entire codebase
  • No SQL injection surface: No raw SQL queries; no database queries in the server
  • No command injection surface: No child_process usage
  • XSS mitigation: React JSX auto-escapes text content; server sanitises chat messages
  • Zod validation: Shared package uses Zod schemas for game data validation with enum constraints

Fixes Applied

1. Password Hashing (Critical)

Vulnerability: Lobby passwords stored and compared as plain text.

Fix: Introduced hashPassword() and verifyPassword() using Node.js crypto.scryptSync with 16-byte random salt and timingSafeEqual for constant-time comparison.

Files: packages/server/src/game/GameSessionManager.ts, packages/server/src/network/socketManager.ts

2. CORS Hardening & Per-Route Rate Limiting (High)

Vulnerability: Fastify CORS set to origin: true in dev, allowing all origins. /api/saves endpoint shared global rate limit.

Fix: Replaced permissive CORS with explicit /^https?:\/\/localhost(:\d+)?$/ and 127.0.0.1 patterns. Added per-route rate limit of 10 req/min on POST /api/saves.

Files: packages/server/src/main.ts

3. Configurable Server URL (High)

Vulnerability: Hardcoded http://localhost:3000 prevented HTTPS/WSS in production.

Fix: Server URL now reads from VITE_SERVER_URL environment variable with localhost fallback.

Files: packages/client/src/ui/screens/MultiplayerLobbyScreen.tsx

4. Typed Espionage Events (Medium)

Vulnerability: EspionageResultEvent.espionageEvent typed as unknown, bypassing compile-time safety.

Fix: Replaced with proper EspionageEvent union type from engine/espionage.ts.

Files: packages/shared/src/types/events.ts

5. Insecure RNG Warning (Medium)

Vulnerability: Math.random() fallback in ID generation silently produced non-cryptographic output.

Fix: Added console.warn when the fallback path is taken, alerting developers to the degraded security posture.

Files: packages/shared/src/utils/id.ts
